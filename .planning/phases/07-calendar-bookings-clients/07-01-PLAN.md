  **In src/routes/admin/bookings.js**, add a GET / route BEFORE the existing PATCH /:id/status route:

  ```
  GET /api/admin/bookings
  Query params (all optional):
    - date: YYYY-MM-DD string (filters bookings where startTime falls on that date)
    - professionalId: UUID string
    - status: one of PRE_RESERVED, CONFIRMED, COMPLETED, CANCELLED, NO_SHOW
  ```

  Implementation:
  - Import prisma directly (like the dashboard route does — no need to go through bookingService for a simple list query)
  - Build a `where` object conditionally based on provided query params
  - For date filter: convert to start-of-day and end-of-day UTC timestamps, filter `startTime: { gte, lte }`
  - Include relations: `client: { select: { id, name, phone } }`, `service: { select: { id, name, durationMin } }`, `professional: { select: { id, name } }`
  - Order by `startTime asc`
  - Validate query with Zod: date as optional regex, professionalId as optional UUID, status as optional enum
  - Return `{ data: bookings }`

  **In src/routes/admin/clients.js**, add three new routes after the existing GET / list route:

  1. `GET /by-phone/:phone` — Validate phone (string min 8 max 20). Call `findClientByPhone(phone)` from clientService. Return `{ data: client }`. This mirrors the apiKey-protected route but under adminAuth.

  2. `POST /` — Validate body `{ name: string min 2 max 200, phone: string min 8 max 20, email?: string email }`. Call `createClient(body)` from clientService. Return 201 `{ data: client }`.

  3. `GET /:id/appointments` — Validate id as UUID. Call `getClientAppointments(id)` from clientService. Return `{ data: appointments }`.

  Import `findClientByPhone`, `createClient`, `getClientAppointments` from clientService (they already exist, just add them to the import destructuring).

  IMPORTANT: Place the `/by-phone/:phone` route BEFORE `/:id/appointments` to avoid Express treating "by-phone" as an :id param.
