  <name>Task 1: Build Calendar page with FullCalendar, booking display, and status transitions</name>
  <files>frontend/src/pages/admin/Calendar.jsx</files>
  <action>
  Create `frontend/src/pages/admin/Calendar.jsx` with:

  **Imports:**
  - FullCalendar from `@fullcalendar/react`, dayGridPlugin from `@fullcalendar/daygrid`, timeGridPlugin from `@fullcalendar/timegrid`, interactionPlugin from `@fullcalendar/interaction`
  - useQuery, useMutation, useQueryClient from @tanstack/react-query
  - useState from react
  - apiFetch from @/lib/api
  - Dialog components from @/components/ui/dialog
  - Button, Badge from @/components/ui/button and @/components/ui/badge
  - toast from sonner

  **Status color map** (object):
  ```
  PRE_RESERVED: { bg: '#fbbf24', border: '#f59e0b', text: '#78350f' }  // yellow
  CONFIRMED:    { bg: '#3b82f6', border: '#2563eb', text: '#ffffff' }  // blue
  COMPLETED:    { bg: '#22c55e', border: '#16a34a', text: '#ffffff' }  // green
  CANCELLED:    { bg: '#9ca3af', border: '#6b7280', text: '#ffffff' }  // gray
  NO_SHOW:      { bg: '#ef4444', border: '#dc2626', text: '#ffffff' }  // red
  ```

  **Data fetching:**
  - Track current date range with useState: `[dateRange, setDateRange]` initialized to `{ start: today ISO date, end: today ISO date }`
  - useQuery with queryKey `['bookings', dateRange.start]` calling `apiFetch('/admin/bookings?date=' + dateRange.start)`
  - Note: For week view, we fetch per-day. Simpler approach: just pass the date of the current view's start. Actually, since the backend filters by single date, and week view shows 7 days, we should fetch ALL bookings for the visible range. Two approaches:
    a) Fetch without date filter (returns all bookings) — fine for small salon
    b) The backend currently only supports single date filter
  - **Use approach (a):** Call `apiFetch('/admin/bookings')` without date filter. This returns all bookings. FullCalendar will position them by their startTime/endTime. For performance, this is fine for a small salon (hundreds of bookings, not thousands). Add a TODO comment noting that date range filtering could be added later for scale.

  **Transform bookings to FullCalendar events:**
  ```js
  const events = (bookings || []).map(b => ({
    id: b.id,
    title: `${b.client?.name || 'Client'} - ${b.service?.name || 'Service'}`,
    start: b.startTime,
    end: b.endTime,
    backgroundColor: STATUS_COLORS[b.status]?.bg,
    borderColor: STATUS_COLORS[b.status]?.border,
    textColor: STATUS_COLORS[b.status]?.text,
    extendedProps: { booking: b },
  }));
  ```

  **FullCalendar component:**
  ```jsx
  <FullCalendar
    plugins={[dayGridPlugin, timeGridPlugin, interactionPlugin]}
    initialView="timeGridDay"
    headerToolbar={{
      left: 'prev,next today',
      center: 'title',
      right: 'timeGridDay,timeGridWeek',
    }}
    events={events}
    eventClick={handleEventClick}
    slotMinTime="07:00:00"
    slotMaxTime="21:00:00"
    allDaySlot={false}
    height="auto"
    locale="pt-br"
    slotDuration="00:30:00"
    nowIndicator={true}
  />
  ```

  Note: Do NOT pass `datesSet` callback to trigger refetch — we fetch all bookings. If we want date-scoped fetching later, add datesSet to update dateRange state.

  **Booking detail dialog:**
  - State: `[selectedBooking, setSelectedBooking]` = null
  - `handleEventClick(info)`: set selectedBooking from `info.event.extendedProps.booking`
  - Dialog shows: client name, phone, service name, duration, professional name, date/time, current status badge
  - Status transition buttons based on current status:
    - PRE_RESERVED: "Confirm" (CONFIRMED), "Cancel" (CANCELLED)
    - CONFIRMED: "Complete" (COMPLETED), "No-Show" (NO_SHOW), "Cancel" (CANCELLED)
    - COMPLETED: no transitions (final)
    - CANCELLED: no transitions (final)
    - NO_SHOW: no transitions (final)
  - Use `useMutation` calling `apiFetch('/admin/bookings/' + id + '/status', { method: 'PATCH', body: { status } })`
  - On success: invalidateQueries(['bookings']), toast.success, close dialog
  - On error: toast.error

  **Layout:**
  ```jsx
  <div>
    <h1 className="mb-6 text-2xl font-semibold">Calendar</h1>
    <div className="rounded-xl border bg-card p-4">
      <FullCalendar ... />
    </div>
    <BookingDetailDialog booking={selectedBooking} open={!!selectedBooking} onOpenChange={...} />
  </div>
  ```

  **FullCalendar CSS:** Import `@fullcalendar/core` styles. FullCalendar 6 with the React connector auto-includes base styles. If needed, add a small CSS override in a `<style>` tag or inline styles to ensure the calendar fits the card container. If FullCalendar styles don't load, add `import '@fullcalendar/core/main.css'` — but first try without it as FC6 bundles styles.

  **Status legend:** Add a simple flex row below the calendar title showing colored dots with status labels, so users know what each color means.
