    2. Create a blank migration (the Prisma schema already has the Booking model — no schema.prisma changes needed):
       ```bash
       npx prisma migrate dev --create-only --name add_booking_conflict_index
       ```

    3. Edit the generated migration file. It will be empty (no schema changes). Add this exact DDL:
       ```sql
       -- Partial unique index: only one active booking per (professional, start_time) slot.
       -- CANCELLED, COMPLETED, NO_SHOW bookings are excluded — they don't block future bookings.
       CREATE UNIQUE INDEX "bookings_active_slot_unique"
         ON "bookings" ("professional_id", "start_time")
         WHERE status IN ('PRE_RESERVED', 'CONFIRMED');
       ```

       IMPORTANT: Use snake_case column names (`professional_id`, `start_time`) — these are the actual PostgreSQL column names. Prisma maps camelCase fields to snake_case columns automatically.

    4. Apply the migration:
       ```bash
       npx prisma migrate dev
       ```

    5. Verify the index exists:
       ```bash
       npx prisma db execute --stdin <<< "SELECT indexname, indexdef FROM pg_indexes WHERE tablename = 'bookings' AND indexname = 'bookings_active_slot_unique';"
       ```

    NOTE on Prisma 6 safety: Prisma 6's shadow database diffing does NOT detect partial indexes. It will NOT generate DROP INDEX in subsequent migrations. This is safe.
  </action>

<task type="auto">
  <name>Task 2: Create pure slot generation and timezone helper module</name>
  <files>src/lib/slots.js</files>
  <action>
    Create `src/lib/slots.js` with two exported functions:

    **1. `generateAvailableSlots(workStart, workEnd, durationMin, activeBookings)`**
    - Pure function, no DB calls, no side effects
    - `workStart`, `workEnd`: UTC Date objects (already converted from local time)
    - `durationMin`: integer (service duration in minutes)
    - `activeBookings`: array of `{ startTime: Date, endTime: Date }` — already TTL-filtered
    - Algorithm: iterate cursor from workStart, stepping by durationMin. For each step:
      - Calculate `slotEnd = addMinutes(cursor, durationMin)`
      - If `slotEnd > workEnd` → break (last slot overrun guard)
      - Check overlap: `activeBookings.some(b => b.startTime < slotEnd && b.endTime > cursor)`
      - If not occupied → push `new Date(cursor)` to results
      - Advance cursor by durationMin
    - Returns: `Date[]` of available slot start times
    - Import `addMinutes` from `date-fns`

    **2. `localTimeToUTC(timeStr, dateStr, timezone)`**
    - Converts a "HH:MM" time string + "YYYY-MM-DD" date string to a UTC Date
    - Uses `fromZonedTime` from `date-fns-tz`
    - Build a local Date: `new Date(\`${dateStr}T${timeStr}:00\`)`
    - Call `fromZonedTime(localDate, timezone)` to get UTC equivalent
    - Returns: UTC `Date`
    - CRITICAL: `fromZonedTime` treats its first argument as local time in the given timezone and converts TO UTC. Do NOT use `toZonedTime` here (that goes UTC→local).

    Follow existing code style: ES module exports, JSDoc comments, no default export.
  </action>
