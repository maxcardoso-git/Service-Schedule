    Follow the exact pattern from `src/routes/clients.js`: Router, apiKeyAuth, asyncHandler, validate with Zod schemas.

    Endpoints:

    a) `POST /api/bookings/availability` (SCHD-01, SCHD-06)
    - Validate body: `{ professionalId: z.string().uuid(), serviceId: z.string().uuid(), date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/) }`
    - Call `getAvailableSlots(professionalId, serviceId, date)`
    - Return 200 `{ data: result }`

    b) `POST /` (SCHD-02, INFR-03)
    - Validate body: `{ clientId: z.string().uuid(), professionalId: z.string().uuid(), serviceId: z.string().uuid(), startTime: z.string().datetime(), idempotencyKey: z.string().max(255).optional() }`
    - Call `createPreReservation(req.body)`
    - Return 201 `{ data: booking }`

    c) `PATCH /:id/confirm` (SCHD-03)
    - Validate params: `{ id: z.string().uuid() }`
    - Call `confirmBooking(req.params.id)`
    - Return 200 `{ data: booking }`

    d) `PATCH /:id/cancel` (SCHD-04)
    - Validate params: `{ id: z.string().uuid() }`
    - Call `cancelBooking(req.params.id)`
    - Return 200 `{ data: booking }`

    e) `GET /by-phone/:phone` (SCHD-05)
    - Validate params: `{ phone: z.string().min(8).max(20) }`
    - Call `getBookingsByPhone(req.params.phone)`
    - Return 200 `{ data: bookings }`

    All routes use `apiKeyAuth` middleware (router.use at top). Use `asyncHandler` wrapper for all async handlers. Import all 5 functions from `../services/bookingService.js`.

    - Import `cron` from `node-cron`, `prisma` from `../lib/prisma.js`, `logger` from `../lib/logger.js`
    - Export `startExpiryJob()` function:
      - Schedule `'* * * * *'` (every minute)
      - In the callback: `prisma.booking.updateMany({ where: { status: 'PRE_RESERVED', expiresAt: { lt: new Date() } }, data: { status: 'CANCELLED' } })`
      - If `result.count > 0`, log: `logger.info(\`Expired ${result.count} pre-reservations\`)`
      - Wrap in try/catch, log errors with `logger.error`
