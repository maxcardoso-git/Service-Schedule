<task type="auto">
  <name>Task 1: Create Vite project with all dependencies</name>
  <files>frontend/package.json, frontend/vite.config.js, frontend/jsconfig.json</files>
  <action>
1. From the project root, run:
   ```
   npm create vite@latest frontend -- --template react
   ```
   This creates frontend/ with React + Vite scaffold.

2. cd into frontend/ and install dependencies:
   ```
   npm install
   npm install -D tailwindcss@^4.2.1 @tailwindcss/vite
   npm install react-router-dom@^7 @tanstack/react-query@^5 zustand@^5
   npm install lucide-react class-variance-authority clsx tailwind-merge sonner
   ```

3. Create frontend/vite.config.js (replace the generated one):
   - Import defineConfig from 'vite', react from '@vitejs/plugin-react', tailwindcss from '@tailwindcss/vite', path from 'path'
   - Plugins: [react(), tailwindcss()]
   - resolve.alias: '@' -> path.resolve(__dirname, './src')
   - server.port: 5173
   - server.proxy: '/api' -> { target: 'http://localhost:3100', changeOrigin: true }
   - build.outDir: 'dist'

   NOTE: Use port 3100 for proxy target (matches .env PORT=3100). The VPS port 3150 is the external-facing port.

4. Create frontend/jsconfig.json:
   ```json
   {
     "compilerOptions": {
       "baseUrl": ".",
       "paths": { "@/*": ["./src/*"] }
     }
   }
   ```

5. Delete any generated CSS files (e.g., src/App.css, src/index.css) as we'll replace them with Tailwind setup.

6. Add frontend/dist/ to the root .gitignore if not already there. Also add frontend/node_modules/ if not covered by a global pattern.
  </action>
  <verify>
    Run `cd /Users/maxcardoso/Projects/Service-schedule/frontend && npm run dev -- --host 2>&1 &` then after 3 seconds `curl -s http://localhost:5173 | head -5` to confirm Vite serves HTML. Kill the process after.
  </verify>
  <done>Vite project created with all dependencies installed, proxy configured to backend, path aliases set up.</done>
</task>

<task type="auto">
  <name>Task 2: Tailwind v4 + shadcn/ui setup</name>
  <files>frontend/src/app.css, frontend/src/lib/utils.js, frontend/src/main.jsx, frontend/src/App.jsx, frontend/components.json</files>
  <action>
1. Create frontend/src/app.css with Tailwind v4 CSS-first config:
   - `@import 'tailwindcss';`
   - CSS custom properties for shadcn/ui theme (see research file for full variable list)
   - `@theme inline` block mapping CSS vars to Tailwind theme tokens
   - Include both light theme variables in :root

2. Create frontend/src/lib/utils.js:
   ```javascript
   import { clsx } from 'clsx';
   import { twMerge } from 'tailwind-merge';
   export function cn(...inputs) {
     return twMerge(clsx(inputs));
   }
   ```

3. Run shadcn init from frontend/:
   ```
   npx shadcn@latest init --defaults --force
   ```
   If it prompts, select: no TypeScript, default style, app.css for globals, use CSS variables.

   IMPORTANT: After init, check if components.json was created. If shadcn modifies app.css, restore the Tailwind v4 setup (shadcn may try to add Tailwind v3 directives). Verify app.css still has `@import 'tailwindcss'` not `@tailwind base`.

4. Install initial shadcn components:
   ```
   npx shadcn@latest add button card input label --force
   ```
   If components are generated as .tsx files, rename them to .jsx and remove TypeScript type annotations (`: React.FC`, `: string`, `interface Props {}`, etc.).

5. Update frontend/src/main.jsx:
   - Import React and ReactDOM
   - Import './app.css'
   - Render App component
   - Keep it minimal (no providers here, App.jsx handles those)

6. Create a minimal frontend/src/App.jsx that just renders a test page:

   ```jsx
           <h1 className="text-3xl font-bold">Service Schedule</h1>
           <Button>Test Button</Button>
         </div>
       </div>
     );
   }
   ```
   This verifies Tailwind, shadcn, and path aliases all work together.
