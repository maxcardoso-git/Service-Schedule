<task type="auto">
  <name>Task 1: AdminRole migration + requireRole middleware</name>
  <files>prisma/schema.prisma, src/middleware/auth.js</files>
  <action>
1. Add AdminRole enum to prisma/schema.prisma:
   ```
   enum AdminRole {
     ADMIN
     RECEPTIONIST
   }
   ```

2. Add `role AdminRole @default(ADMIN)` to the AdminUser model. Use default ADMIN so existing records get ADMIN automatically during migration.

3. Run `npx prisma migrate dev --name add-admin-role` to create and apply the migration.

4. AFTER migration succeeds, change the default in schema.prisma from `@default(ADMIN)` to `@default(RECEPTIONIST)` so new users default to receptionist. Run `npx prisma migrate dev --name default-role-receptionist` to record this change.

5. Add `requireRole()` function to src/middleware/auth.js:
   - Import ForbiddenError from '../lib/errors.js'
   - Function accepts spread `...roles` parameter
   - Returns middleware that checks `req.admin.role` against allowed roles
   - Throws ForbiddenError('Insufficient permissions', 'FORBIDDEN') if not authorized
   - Export it as named export alongside existing adminAuth

IMPORTANT: Do NOT change adminAuth. Only ADD requireRole as a new export.
  </action>
  <verify>
    Run `npx prisma migrate status` -- should show all migrations applied.
    Run `node -e "import('./src/middleware/auth.js').then(m => console.log(typeof m.requireRole, typeof m.adminAuth))"` -- should print "function function".
  </verify>
  <done>AdminRole enum exists in schema, role column on AdminUser with RECEPTIONIST default, existing users have ADMIN role, requireRole() middleware exported from auth.js.</done>
</task>
