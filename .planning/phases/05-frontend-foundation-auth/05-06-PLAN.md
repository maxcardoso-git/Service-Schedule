   - Use useQuery to fetch users: queryKey ['admin-users'], queryFn calls `apiFetch('/admin/users')`
   - Display users in a shadcn Table:
     - Columns: Name, Email, Role (Badge component: ADMIN=default, RECEPTIONIST=secondary), Status (Badge: active=green, inactive=gray), Actions
     - Actions column: Edit button (Pencil icon), toggle active switch
   - **Create User Dialog:**
     - Open when "New User" button clicked (useState for open state)
     - Form fields: name (Input), email (Input), password (Input type=password), role (Select with ADMIN/RECEPTIONIST options)
     - Submit button calls useMutation with `apiFetch('/admin/users', { method: 'POST', body: formData })`
     - On success: close dialog, invalidate 'admin-users' query (via queryClient.invalidateQueries), show success toast via sonner
     - On error: show error message in dialog
   - **Edit User Dialog:**
     - Open when Edit button clicked, pre-populated with user data
     - Form fields: name (Input), role (Select), active (Switch)
     - Submit: useMutation with `apiFetch('/admin/users/' + user.id, { method: 'PATCH', body: updates })`
     - On success: close dialog, invalidate queries, show toast
   - **Toggle Active:**
     - Switch in Actions column directly calls PATCH to toggle active status
     - Use useMutation, invalidate queries on success
   - Loading state: show skeleton rows while fetching
   - Empty state: "No users found" message

   - The route should already be wrapped in ProtectedRoute with roles={['ADMIN']} from plan 05

IMPORTANT: Use `data.data` pattern when reading API responses (backend wraps in `{ data: ... }`). So `apiFetch('/admin/users')` returns `{ data: [...users] }`, access users via `response.data`.
