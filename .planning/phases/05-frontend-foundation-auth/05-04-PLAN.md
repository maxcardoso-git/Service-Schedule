<task type="auto">
  <name>Task 1: Auth store + API client</name>
  <files>frontend/src/stores/authStore.js, frontend/src/lib/api.js</files>
  <action>
1. Create frontend/src/stores/authStore.js using Zustand:
   - State: token (read from localStorage on init), user (null initially, object with id/email/name/role)
   - `login(token, admin)`: save token to localStorage, set token + user in state
   - `logout()`: remove token from localStorage, set token=null, user=null
   - `initFromToken()`: read token from localStorage, decode JWT payload via `JSON.parse(atob(token.split('.')[1]))`, check expiry (`payload.exp * 1000 < Date.now()`), if expired call logout, if valid set user from payload fields (id, email, role). Wrap in try/catch -- any decode error calls logout.

   Do NOT use any JWT library. The `atob` + `JSON.parse` approach is sufficient for reading the payload.

2. Create frontend/src/lib/api.js:
   - Define `ApiError` class extending Error with status and code properties
   - Export `apiFetch(path, options = {})`:
     - Read token from `useAuthStore.getState().token`
     - Call `fetch('/api' + path, ...)` with Content-Type: application/json, Authorization: Bearer token (if token exists)
     - If options.body is provided, JSON.stringify it
     - On 401 response: call `useAuthStore.getState().logout()`, set `window.location.href = '/login'`, throw ApiError
     - On other non-ok responses: parse JSON, throw ApiError with status, error.code, error.message from response
     - On success: parse and return JSON

   IMPORTANT: The path parameter should NOT include '/api' prefix -- apiFetch prepends it. So login calls `apiFetch('/admin/auth/login', ...)`.
  </action>

  </verify>
  <done>Auth store manages JWT lifecycle with localStorage persistence, API client handles auth headers and 401 auto-redirect.</done>
</task>
