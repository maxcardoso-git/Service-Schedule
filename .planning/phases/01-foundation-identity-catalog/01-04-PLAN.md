  <name>Task 1: Service and professional business logic</name>
  <files>
    src/services/serviceService.js
    src/services/professionalService.js
  </files>
  <action>
    1. Create `src/services/serviceService.js` with exported functions:

       a. `listActiveServices()`: `prisma.service.findMany({ where: { active: true }, select: { id: true, name: true, description: true, durationMin: true, price: true }, orderBy: { name: 'asc' } })`. Return array.

       b. `getServiceById(id)`: `prisma.service.findUnique({ where: { id }, include: { professionals: { include: { professional: { select: { id: true, name: true, active: true } } } } } })`. If not found, throw NotFoundError('Service not found', 'SERVICE_NOT_FOUND'). Filter professional list to active only in response.

       c. `createService({ name, description, durationMin, price })`: `prisma.service.create(...)`. Return created service.

       d. `updateService(id, data)`: Find first, throw NotFoundError if missing. `prisma.service.update({ where: { id }, data })`. Return updated service. Only allow updating: name, description, durationMin, price.

       e. `deactivateService(id)`: Find first, throw NotFoundError if missing. `prisma.service.update({ where: { id }, data: { active: false } })`. Return updated service.

    2. Create `src/services/professionalService.js` with exported functions:

       a. `getProfessionalById(id)`: findUnique with include for services (with service details) and workingHours. Throw NotFoundError if missing.

       b. `createProfessional({ name, email, phone })`: prisma.professional.create. Return created.

       c. `updateProfessional(id, data)`: Find first, update name/email/phone/active. Return updated.

       d. `assignService(professionalId, serviceId)`: Verify both professional and service exist (throw NotFoundError for each). Create ProfessionalService record. Prisma P2002 on duplicate will be caught by errorHandler as 409.

       e. `removeService(professionalId, serviceId)`: Find the ProfessionalService record by unique constraint [professionalId, serviceId]. If not found, throw NotFoundError. Delete it.

       f. `replaceWorkingHours(professionalId, hours)`: Verify professional exists. `hours` is an array of `{ dayOfWeek: number, startTime: string, endTime: string }`. Use a transaction: delete all existing WorkingHours for this professional, then createMany with new hours. This is a full replace (not upsert) — simpler and prevents orphan records.
         - Validate: dayOfWeek 0-6, startTime/endTime format "HH:MM" (24h), startTime < endTime.
         - Do validation in the service layer before the transaction.

    Import prisma, NotFoundError, ValidationError.
  </action>
  <verify>
    `node -e "import('./src/services/serviceService.js').then(m => console.log(Object.keys(m)))"` prints function names.
    `node -e "import('./src/services/professionalService.js').then(m => console.log(Object.keys(m)))"` prints function names.
  </verify>
  <done>
    Service and professional business logic handles all CRUD operations, service assignments, and working hours replacement with proper validation and error handling.
  </done>

  <action>
    1. Create `src/routes/services.js` (public, API key auth) as an Express Router:
       - Apply `apiKeyAuth` middleware to entire router.
       - `GET /` — list active services. Return `{ data: services }`.
       - `GET /:id` — get service by ID with professionals. Validate `id: z.string().uuid()`. Return `{ data: service }`.
       Use asyncHandler wrapper pattern from Plan 03.

    2. Create `src/routes/admin/services.js` (admin, JWT auth) as an Express Router:
       - Apply `adminAuth` middleware to entire router.
       - `POST /` — create service. Validate body: `name: z.string().min(2).max(200), description: z.string().optional(), durationMin: z.number().int().min(5).max(480), price: z.number().positive()`. Return 201 with `{ data: service }`.
       - `PUT /:id` — update service. Same body schema but all fields optional (z.object partial). Return `{ data: service }`.
       - `PATCH /:id/deactivate` — deactivate service. Return `{ data: service }`.

    3. Create `src/routes/admin/professionals.js` (admin, JWT auth) as an Express Router:
       - Apply `adminAuth` middleware to entire router.
       - `GET /:id` — get professional by ID with services and working hours. Return `{ data: professional }`.
       - `POST /` — create professional. Validate body: `name: z.string().min(2).max(200), email: z.string().email().optional(), phone: z.string().min(8).max(20).optional()`. Return 201 with `{ data: professional }`.
       - `PUT /:id` — update professional. Partial body schema. Return `{ data: professional }`.
       - `POST /:id/services` — assign service. Validate body: `serviceId: z.string().uuid()`. Return 201 with `{ data: assignment }`.
       - `DELETE /:id/services/:serviceId` — remove service assignment. Return `{ data: { message: 'Service removed from professional' } }`.
       - `PUT /:id/working-hours` — replace working hours. Validate body: `hours: z.array(z.object({ dayOfWeek: z.number().int().min(0).max(6), startTime: z.string().regex(/^\d{2}:\d{2}$/), endTime: z.string().regex(/^\d{2}:\d{2}$/) }))`. Return `{ data: { professionalId, hours: updatedHours } }`.

    4. Update `src/app.js`:
       - Mount public services: `app.use('/api/services', servicesRouter)`.
       - Mount admin services: `app.use('/api/admin/services', adminServicesRouter)`.
       - Mount admin professionals: `app.use('/api/admin/professionals', adminProfessionalsRouter)`.
       - Keep error handler as LAST middleware.
