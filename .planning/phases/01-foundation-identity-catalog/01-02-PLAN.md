  <name>Task 1: Create middleware (auth, validation, error handler) and health check</name>
  <files>
    src/middleware/auth.js

    src/middleware/errorHandler.js
    src/routes/health.js
  </files>
  <action>
    1. Create `src/middleware/auth.js` with TWO exported middleware functions:

       a. `apiKeyAuth(req, res, next)`: Reads `x-api-key` header (case-insensitive — Express lowercases headers). Compares against `process.env.API_KEY`. If missing or invalid, throw `new UnauthorizedError('Invalid or missing API key', 'INVALID_API_KEY')`. If valid, call `next()`.

       b. `adminAuth(req, res, next)`: Reads `Authorization` header, expects `Bearer <token>`. Verify JWT using `jsonwebtoken.verify(token, process.env.JWT_SECRET)`. Attach decoded payload to `req.admin = decoded`. If missing/invalid/expired, throw `new UnauthorizedError('Invalid or expired token', 'INVALID_TOKEN')`.

    2. Create `src/middleware/validate.js` exporting a `validate(schema)` factory function:
       - `schema` is an object with optional keys: `body`, `query`, `params` — each a Zod schema.
       - Returns middleware that validates `req.body`, `req.query`, `req.params` against corresponding schemas.
       - On success: replaces `req.body`/`req.query`/`req.params` with parsed (coerced/transformed) values, calls `next()`.
       - On failure: throw `new ValidationError('Validation failed', zodError.flatten().fieldErrors)`.
       - IMPORTANT: Use `safeParse`, not `parse` — catch errors gracefully.

    3. Create `src/middleware/errorHandler.js` exporting `errorHandler(err, req, res, next)`:
       - If `err instanceof AppError`: respond with `err.statusCode` and body `{ error: { code: err.code, message: err.message, details: err.details || undefined } }`.
       - If Prisma known error (P2002 unique constraint): respond 409 with `{ error: { code: 'CONFLICT', message: 'Resource already exists' } }`.
       - Otherwise: log error with logger, respond 500 with `{ error: { code: 'INTERNAL_ERROR', message: 'Internal server error' } }`. Never leak stack traces in production.

    4. Create `src/routes/health.js` as an Express Router:
       - `GET /` — returns `{ status: 'ok', timestamp: new Date().toISOString(), database: 'connected' | 'disconnected' }`.
       - Check database by running `prisma.$queryRaw\`SELECT 1\``. If it fails, set database to 'disconnected' but still return 200 (health check should not crash).

    5. Update `src/app.js`:
       - Mount health route: `app.use('/api/health', healthRouter)` — health is NOT behind API key auth.
       - Add error handler as LAST middleware: `app.use(errorHandler)`.
       - Do NOT apply apiKeyAuth globally — it will be applied per-route in Plans 03/04.
  </action>
  <verify>
    - Start server: `node src/server.js &`
    - `curl http://localhost:3100/api/health` returns 200 with `{ "status": "ok", "database": "connected" }`
    - `curl -X GET http://localhost:3100/api/nonexistent` returns 404 (Express default, or custom if you add a catch-all)
    - Kill server process.
  </verify>
  <done>
    API key middleware rejects requests without valid key. JWT middleware validates admin tokens. Zod validation middleware returns field-level errors. Error handler produces structured `{ error: { code, message, details? } }` responses. Health check confirms database connectivity.
  </done>

    1. Create `src/routes/admin/auth.js` as an Express Router:
       - `POST /login` — accepts `{ email, password }` validated with Zod schema (email: z.string().email(), password: z.string().min(1)).
       - Query `prisma` for an AdminUser by email (see step 3 below for AdminUser model — if not in schema, store admin credentials in a simple JSON config or add AdminUser model).
       - DECISION: Since the Prisma schema from Plan 01 does NOT include an AdminUser model, add one now. Add to `prisma/schema.prisma`:
         ```
         model AdminUser {
           id        String   @id @default(uuid()) @db.Uuid
           name      String   @db.VarChar(200)
           email     String   @unique @db.VarChar(255)
           password  String   @db.VarChar(255) // bcrypt hash
           active    Boolean  @default(true)
           createdAt DateTime @default(now()) @db.Timestamptz
           updatedAt DateTime @updatedAt @db.Timestamptz

           @@map("admin_users")
         }
         ```
       - Run `npx prisma db push` after adding the model.
       - Compare password with bcryptjs: `bcrypt.compare(password, user.password)`.
       - If valid: sign JWT with `{ id: user.id, email: user.email, role: 'admin' }`, expiresIn `'8h'`, using `process.env.JWT_SECRET`. Return `{ token, admin: { id, name, email } }`.
       - If invalid email or password: throw `UnauthorizedError('Invalid credentials', 'INVALID_CREDENTIALS')`.

    2. Mount in `src/app.js`: `app.use('/api/admin/auth', adminAuthRouter)`. Admin auth route is NOT behind API key auth (it IS the auth mechanism).

    3. Create `prisma/seed.js` that populates a beauty salon demo:

       - Professional-service assignments: Ana does Corte Feminino + Coloracao + Escova Progressiva. Carlos does Corte Masculino. Maria does Manicure + Corte Feminino.
       - Working hours for each professional: Mon-Fri 09:00-18:00, Sat 09:00-14:00. No Sunday.
       - Use `prisma.xxx.upsert()` for idempotent seeding (safe to run multiple times).
       - Use `import { PrismaClient } from '@prisma/client'` and `import bcrypt from 'bcryptjs'`.

    4. Add seed config to `package.json`:
