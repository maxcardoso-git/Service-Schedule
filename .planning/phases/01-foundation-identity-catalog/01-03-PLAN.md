<task type="auto">
  <name>Task 1: Client service layer</name>
  <files>src/services/clientService.js</files>
  <action>
    Create `src/services/clientService.js` with three exported async functions:

    1. `findClientByPhone(phone)`:
       - Normalize phone: strip all non-digit characters, trim.
       - Query `prisma.client.findUnique({ where: { phone } })`.
       - If not found, throw `new NotFoundError('Client not found', 'CLIENT_NOT_FOUND')`.
       - Return client object.

    2. `createClient({ name, phone, email })`:
       - Normalize phone same way.
       - `prisma.client.create({ data: { name, phone, email } })`.
       - Prisma will throw P2002 on duplicate phone — let it bubble to errorHandler (which already handles P2002 as 409 CONFLICT).
       - Return created client.

    3. `getClientAppointments(clientId)`:
       - First verify client exists: `prisma.client.findUnique({ where: { id: clientId } })`. If not found, throw NotFoundError.
       - Query `prisma.booking.findMany({ where: { clientId }, include: { services: { include: { service: true } }, professional: true }, orderBy: { startTime: 'desc' } })`.
       - Return array (may be empty — that is valid).

    Import prisma from '../lib/prisma.js', import NotFoundError from '../lib/errors.js'.
  </action>
  <verify>
    `node -e "import('./src/services/clientService.js').then(m => console.log(Object.keys(m)))"` prints ["findClientByPhone", "createClient", "getClientAppointments"].
  </verify>
  <done>
    Service layer handles phone normalization, client lookup, creation, and appointment history with proper error throwing.
  </done>
</task>

       - `POST /`:
         - Validate body with Zod: `name: z.string().min(2).max(200), phone: z.string().min(8).max(20), email: z.string().email().optional()`.
         - Call `createClient(req.body)`.
         - Return 201 with `{ data: client }`.

       - `GET /:id/appointments`:
         - Validate params: `id: z.string().uuid()`.
         - Call `getClientAppointments(req.params.id)`.
         - Return 200 with `{ data: appointments }`.

       Use the `validate` middleware from Plan 02 for all Zod validation.
       Wrap all route handlers in try/catch and call `next(error)` on catch — OR use a small async wrapper: `const asyncHandler = (fn) => (req, res, next) => Promise.resolve(fn(req, res, next)).catch(next)`.
