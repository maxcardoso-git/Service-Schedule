**Tagging convention:**
- `Health` tag: health.js
- `Clients` tag: clients.js
- `Services` tag: services.js
- `Bookings` tag: bookings.js
- `Payments` tag: payments.js

**Endpoints to annotate (10 total across 5 files):**

health.js:
- `GET /api/health` — no security, returns `{ status, timestamp, database }`

clients.js:
- `GET /api/clients/by-phone/:phone` — path param phone, returns client object or 404
- `POST /api/clients` — body `{ name, phone, email? }`, returns 201 with client, 409 on duplicate phone
- `GET /api/clients/:id/appointments` — path param id (uuid), returns array of bookings

services.js:
- `GET /api/services` — no params, returns array of active services with professionals

bookings.js:
- `GET /api/bookings?conversationId=` — query param conversationId (required), returns array of bookings. 400 if missing.
- `POST /api/bookings/availability` — body `{ professionalId, serviceId, date }`, returns availability object
- `POST /api/bookings` — body `{ clientId, professionalId, serviceId, startTime, idempotencyKey?, conversationId? }`, returns 201 booking
- `PATCH /api/bookings/:id/confirm` — path param id, returns confirmed booking
- `PATCH /api/bookings/:id/cancel` — path param id, returns cancelled booking
- `GET /api/bookings/by-phone/:phone` — path param phone, returns bookings array

payments.js:
- `POST /api/payments/pix` — body `{ bookingId }`, returns 201 payment with pixPayload
- `GET /api/payments/:id/status` — path param id, returns payment status object
- `POST /api/payments/:id/simulate-paid` — path param id, returns updated payment

**Response schema rules:**
- All success responses wrap in `{ data: ... }` envelope
- All error responses use `{ error: { code: string, message: string } }` envelope
- Use `$ref: '#/components/responses/ValidationError'` for 400 responses
- Use `$ref: '#/components/responses/NotFound'` for 404 responses
- Use `$ref: '#/components/responses/Conflict'` for 409 responses
- Health endpoint has `security: []` to override global ApiKeyAuth

**Do NOT change any route handler logic** — only add JSDoc comment blocks above the existing handlers.
